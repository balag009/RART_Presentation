Project Overview

The Challenge

Patients often face significant delays in understanding their symptoms, identifying the right specialist, and booking timely appointments. This leads to increased anxiety, potential worsening of conditions, and an inefficient healthcare system burden.

  • Lack of immediate, personalized symptom assessment.
  • Difficulty navigating complex specialist referrals.
  • Inefficient and time-consuming appointment booking.
  • Critical need for human oversight in AI-driven healthcare decisions.

Our Solution: RART

RART is an intelligent, AI-powered healthcare assistant designed to streamline the patient journey from initial symptom analysis to personalized treatment planning and efficient appointment booking. A crucial component of RART is its integrated human review workflow, ensuring that every AI-driven suggestion is validated by a medical professional.

How RART Works: Workflow

The core of RART's operation is a sophisticated, AI-driven workflow managed by LangGraph, ensuring a seamless and intelligent patient journey from initial symptoms to comprehensive care.

Figure: Detailed Workflow of the RART Application, showing agent interactions and decision points.

Workflow Steps:

  1. Patient Input: Patient describes symptoms, provides name and email via the Streamlit user interface.
  2. Symptom Analysis (AI): The Symptom Analyzer Agent (using Google Gemini) determines symptom severity and suggests a specialist. If critical symptoms are detected, the system advises immediate emergency care and the process concludes without automated booking.
  3. Treatment Planning (AI): The Treatment Planning Agent generates a personalized general treatment plan based on the AI's initial assessment.
  4. Human Review: The system routes to the Human Review Agent, presenting the AI's suggestions to a medical professional. Here, they can review, modify, or reject the AI's assessment and treatment plan.
  5. Appointment Booking: If the AI's suggestions are approved (or modified and approved) by the human reviewer, the Appointment Booking Agent attempts to secure an appointment with the suggested specialist.
  6. Email Confirmation: The Email Sender Agent dispatches a comprehensive email to the patient, including appointment details and the reviewed treatment plan.
  7. Completion: The process concludes, providing the patient with clear next steps and confirmation.

Key Workflow Integrations:

  • LangGraph Orchestration: The entire multi-step process is seamlessly orchestrated by LangGraph, managing state transitions and agent calls.
  • Agent-Tool Interaction: Each agent leverages specific tools (e.g., `gemini_api_tool`, `booking_simulator_tool`, `email_sender_tool`) to perform its designated tasks.
  • Human-in-the-Loop: The human review step is a critical control point, ensuring accuracy and ethical guidelines are met before patient communication.
  • Conditional Routing: The workflow dynamically adapts based on AI analysis (e.g., critical vs. non-critical symptoms) and human review decisions (e.g., approval vs. rejection).

RART System Architecture

RART's robust architecture is designed for modularity, scalability, and secure operation, integrating AI capabilities with essential human oversight.

Figure: Architectural Overview of RART's Components and Interactions.

Key Architectural Layers:

  • User Interface Layer: Developed with Streamlit (`app.py`), serving as the primary interaction point for both patients (input) and medical professionals (review portal).
  • Core Workflow Orchestration: Powered by LangGraph (`workflow_builder.py`), it defines and manages the multi-step execution flow, using `AgentState` (`agent_state.py`) for shared data.
  • AI Agents Layer: Comprises specialized Python functions (e.g., `symptom_analyzer_agent.py`, `treatment_planning_agent.py`, `human_review_agent.py`, `appointment_booking_agent.py`, `email_sender_agent.py`) acting as nodes in the LangGraph, each responsible for a specific process step.

Interactions & External Services:

  • Tools Layer: Encapsulates specific functionalities as LangChain Tools (e.g., `gemini_api_tool.py`, `booking_simulator_tool.py`, `email_sender_tool.py`), callable by the AI agents.
  • External Services:
    • Google Gemini API: Provides the core AI intelligence for symptom analysis and treatment plan generation.
    • SMTP Server: Used for secure email communication.
    • `.env` File: Manages sensitive environment variables securely using `python-dotenv`.
  • Data Flow: Information (patient symptoms, AI assessments, human decisions) flows through the `AgentState`, enabling seamless communication between all components.

Technology Stack

RART is built on a modern and robust technology stack, leveraging leading frameworks for AI, workflow automation, and user interface development.

AI / LLM Integration

  • Google Gemini: Core Large Language Model for intelligent analysis and generation.
  • `google-generativeai`: Python client library for interacting with Gemini API.

Orchestration & Agents

  • LangChain: Foundational framework for LLM-powered applications.
  • LangGraph: For defining and managing complex, stateful agentic workflows.

User Interface

  • Streamlit: For building fast, interactive web applications and the human review portal.
  • Tailwind CSS (via CDN): For modern, utility-first styling.

Data & Environment

  • Python Standard Libraries (e.g., `datetime`, `json`): For various utilities.

Communication

  • `smtplib` (Python): For sending email notifications.
  • Custom LangChain Tools: Encapsulating external interactions (e.g., email sending, booking simulation).

Development & Deployment

  • Python: Primary programming language.
  • `requirements.txt`: Manages project dependencies.
  • Modular File Structure: Agents, tools, workflow in separate files for maintainability.
